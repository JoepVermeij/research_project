# VIII. Verantwoording AI

### Alessio Veppi
Voor de ontwikkeling van de Proof of Concept (POC) heb ik Claude Code (claude-opus-4-7) ingezet als mentor en reviewer. Na het omzetten van de research paper naar Markdown, fungeerde dit document als vaste context voor het model. Ik heb de broncode grotendeels zelf geschreven, waarna Claude feedback gaf. Slechts bij twee complexe deelproblemen heb ik code laten genereren: de AES-GCM 256-bit encryptie via de Web Crypto API en het ververs-mechanisme van de dynamische QR-tokens. Deze gegenereerde code is achteraf grondig bestudeerd en handmatig geïntegreerd, zodat alle technische keuzes volledig verantwoord zijn.

### Joris Dreesen
Voor het literatuuronderzoek heb ik gebruikgemaakt van Perplexity AI om efficiënt academische bronnen te verzamelen. Via gerichte prompts heb ik het model de opdracht gegeven om peer-reviewed artikelen te zoeken omtrent fraudebestrijding bij QR-registraties. Daarbij heb ik expliciet gevraagd om de resultaten te structureren met een APA-referentie, een samenvatting en een toepassing van de CRAAP-test (Currency, Relevance, Authority, Accuracy, Purpose). Vanuit deze gestructureerde output heb ik vervolgens handmatig de meest betrouwbare en relevante bronnen geselecteerd voor onze uiteindelijke bronbespreking.

### Joep Vermeij
Voor het redigeren van de paper en het structureren van de POC heb ik gebruikgemaakt van Gemini. Mijn werkwijze bestond uit een iteratieve "vraag, genereer, review"-methode. In plaats van de tekst volledig zelf uit te schrijven, liet ik het model oplossingen genereren op basis van gerichte vragen en prompts. Deze output heb ik vervolgens kritisch gereviewd en aangepast aan onze projectcontext. Omdat het model sterk is in formatteren, maar ik het bredere overzicht bezit, bleek deze iteratieve wisselwerking ideaal om alle documentatie snel, uniform en professioneel te stroomlijnen.
